# 04 -- Bento Card Grid

## Design Philosophy

Modern SaaS feel built around bento-style cards with hover states and micro-interactions. Information density done right -- every card earns its space in the grid, and interactivity rewards exploration without overwhelming the user. The design communicates technical competence through clean geometry, consistent spacing, and smooth motion.

## UX Pattern

The user lands on a clean centered hero and scrolls into a bento grid of feature cards with varying sizes (1x1, 2x1, 1x2). Cards stagger in as the viewport reaches them, creating a cascading entrance effect. Hovering a card lifts it with an enhanced shadow and reveals hidden content -- a description, a secondary CTA, or additional details -- via a slide-up overlay.

Below the features, a muted stats section runs counter animations on scroll. The portfolio section offers a filterable card grid where selecting a category smoothly rearranges cards with layout transitions. Testimonials sit in consistent card containers with avatars. The pricing section uses tiered cards with a monthly/annual toggle that smoothly transitions between values. The page closes with a single accent CTA.

Seven sections total: hero, bento features, stats, filterable portfolio, card testimonials, pricing tiers, and accent CTA.

## When to Use

- SaaS products that need to communicate many features without feeling cluttered
- Tech startups and developer tools where the audience expects modern, polished UI
- Dashboard or analytics products where data density is a selling point
- Any product-led brand that wants to feel current and technically sharp
- Teams that want rich interactivity without sacrificing load performance

## When NOT to Use

- Brands that need a warm, human, or organic feel -- this template reads as "tech product"
- Content-heavy editorial sites where long-form text is the primary format
- Luxury or premium brands where minimalism and whitespace are critical
- Clients with very few features or offerings -- the bento grid needs at least 4-5 items to look right
- Projects targeting audiences unfamiliar with modern SaaS UI conventions
